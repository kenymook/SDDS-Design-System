# Как выбрать компонент

В SDDS ~60 компонентов, и для многих задач есть несколько подходящих. Ниже — таблицы решений по типичным сценариям. Если вашей задачи нет — посмотрите [полный каталог](../reference/components.md).

---

## Уведомления и сообщения

| Задача | Компонент | Почему |
|---|---|---|
| Системное уведомление, исчезает само | `Toast` | Временное, не требует реакции |
| Важное сообщение, требует действия | `Notification` | Остаётся до взаимодействия |
| Подсказка при наведении / фокусе | `Tooltip` | Только текст, без действий |
| Контекстный popup с действиями | `Popover` | Произвольный контент, интерактивен |
| Заметка в потоке контента | `Note` | Статичная, не overlay |

**Правило:** если сообщение требует реакции пользователя — это не Toast. Toast пропадёт раньше, чем пользователь среагирует.

## Выбор из списка

| Задача | Компонент | Почему |
|---|---|---|
| Один из ≤5 видимых вариантов | `Segment` | Все варианты на виду, компактно |
| Один из ≤5, требуется визуальный вес | `RadioBox` | Стандартный паттерн |
| Один из многих | `Select` (Single) | Dropdown |
| Несколько из многих | `Select` (Multiple) | Dropdown с чекбоксами |
| Свободный ввод + подсказки | `Autocomplete` | Можно ввести что угодно |
| Только из списка, ввод фильтрует | `ComboBox` | Жёстко из списка, ввод — поиск |
| Иерархия (категории, папки) | `TreeSelect` | Многоуровневая структура |
| Бинарный выбор | `CheckBox` или `Switch` | См. ниже |

### CheckBox vs Switch

| | `CheckBox` | `Switch` |
|---|---|---|
| Применяется к данным | Да (значение в форме) | Нет (мгновенное действие) |
| Требует подтверждения | Submit | Нет, действие моментальное |
| Состояние | `checked` / `unchecked` / `indeterminate` | `on` / `off` |
| Пример | «Я согласен с условиями» | «Включить уведомления» |

Если изменение применяется только после нажатия Submit — это CheckBox. Если эффект мгновенный — Switch.

## Кнопки

| Задача | Компонент |
|---|---|
| Основное действие в форме / на странице | `BasicButton` `View=Accent` |
| Деструктивное действие (удалить, отменить заказ) | `BasicButton` `View=Negative` |
| Вторичное действие («Отмена» рядом с «Сохранить») | `BasicButton` `View=Default` или `Secondary` |
| Кнопка только с иконкой | `IconButton` (всегда с `aria-label`) |
| Действие, выглядящее как ссылка | `LinkButton` |
| Кнопка внутри другого компонента (Toast, Notification) | `EmbeddedButton` |
| Группа связанных действий | `ButtonGroup` |

**На странице — одна основная кнопка.** Если их две `View=Accent` рядом, пользователь не понимает, какая важнее.

## Панели и слои

| Задача | Компонент | Почему |
|---|---|---|
| Блокирующий диалог (требует ответа) | `Modal` | Останавливает работу, требует действия |
| Боковая панель (фон остаётся доступен) | `Drawer` | Можно работать с фоном параллельно |
| Панель снизу (mobile) | `BottomSheet` | Touch-паттерн |
| Контекстные действия над выделением | `ToolBar` | Появляется при контексте |

### Modal vs Drawer

Главный вопрос: **должен ли пользователь сначала закрыть панель, чтобы продолжить работу?**

- Да → `Modal` (блокирует)
- Нет → `Drawer` (фон активен)

Модалы используются реже, чем кажется. Большинство «модалок» в продуктах должны быть Drawer'ами.

## Статусы и метки

| Задача | Компонент | Почему |
|---|---|---|
| Числовой счётчик (5 уведомлений) | `Badge` | Компактно, без действий |
| Тег с возможностью удаления | `Chip` | `hasClose=True`, интерактивен |
| Статус записи (активен, заблокирован) | `Badge` `View=Positive/Negative/Warning` | Семантический цвет |
| Онлайн-индикатор у аватара | `Avatar` + `Status` | Встроено в компонент |
| Точечный индикатор состояния | `Indicator` | Минимальная метка |

### Badge vs Chip

| | `Badge` | `Chip` |
|---|---|---|
| Интерактивен | Нет | Да |
| Можно удалить | Нет | Да (`hasClose`) |
| Может быть ссылкой | Нет | Да |
| Пример | Счётчик уведомлений | Фильтр в поиске |

Если пользователь должен иметь возможность нажать или удалить элемент — это Chip.

## Загрузка и прогресс

| Задача | Компонент |
|---|---|
| Неопределённое ожидание (ждём ответа) | `Spinner` |
| Прогресс с известным процентом | `Loader` или `ProgressBar` |
| Прогресс многошаговой операции | `Steps` |

`Spinner` — для коротких ожиданий (< 5 сек). Для долгих операций используйте `ProgressBar` — иначе пользователь думает, что приложение зависло.

## Навигация

| Задача | Компонент |
|---|---|
| Переключение между секциями страницы | `Tabs` |
| Цепочка положения в иерархии | `BreadCrumbs` |
| Постраничный список | `Pagination` |
| Точечная навигация (карусель) | `PaginationDots` |
| Многошаговый процесс (онбординг, мастер) | `Steps` |
| Нижняя навигация (mobile) | `TabBar` |
| Меню действий по клику | `DropdownMenu` |
| Онбординг-тур | `Tour` |

### Tabs vs Pagination

- `Tabs` — все секции существуют параллельно, пользователь переключает фокус.
- `Pagination` — секции отображаются последовательно, пользователь движется по списку.

«Профиль / Настройки / Безопасность» — Tabs (это разные секции одной сущности).
«Страница 1, 2, 3 ...» — Pagination.

## Ввод дат и времени

| Задача | Компонент |
|---|---|
| Одна дата | `DatePicker` |
| Дата + время | `DateTimePicker` |
| Только время | `TimePicker` |
| Версия с возможностью очистки | `*Clear` (например, `DatePickerClear`) |

`*Clear`-варианты добавляют кнопку очистки в `contentRight`. Используйте их, если поле необязательное — пользователь должен иметь возможность отменить выбор.
