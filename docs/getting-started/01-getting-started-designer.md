# Старт для дизайнера

После этого туториала вы:

- открыли SDDS в Figma и поняли, что где лежит
- собрали форму из готовых компонентов
- переключили тему и увидели, что компоненты адаптировались сами
- знаете, куда смотреть дальше

Время: 15 минут.

---

## Шаг 1. Откройте библиотеку

1. В Figma → **Assets** → найдите библиотеку **SDDS**.
2. Если её нет в списке — попросите Figma-админа команды дать доступ.
3. Откройте файл `SDDS | Styles` (fileKey `0FxQGHmGUOCjtHM3N9j4Oq`) — это исходник системы.

В файле три ключевых места:

| Где | Что |
|---|---|
| **Pages** в левой панели | Компоненты по категориям: Data Display, Data Entry, Overlay, Navigation |
| **Local Variables** в правой панели | Все цвета и числа системы, сгруппированные в коллекции |
| **Assets** | То, что вы будете перетаскивать в свои макеты |

> **Термин:** *Variables* — переменные Figma, в которых хранятся цвета и числа системы. Подмена коллекции = смена темы. [Подробнее →](../concepts/architecture.md)

---

## Шаг 2. Найдите компонент по задаче

Откройте каталог: **Pages → Data Entry → BasicButton**.

Внутри — несколько вариантов кнопки. Каждый размер обозначен дважды:

```
BasicButton 48 M
            ↑   ↑
            px  T-shirt size
```

`M` (48px) — дефолтный размер. Используйте его, если нет причин отклоняться.

Перетащите `BasicButton 48 M` в свой макет.

---

## Шаг 3. Поменяйте вариант кнопки

Выделите кнопку в макете. В правой панели появятся пропы:

| Проп | Что меняет |
|---|---|
| `View` | Семантика: `Default`, `Accent`, `Negative`, `Positive`, `Warning` |
| `Stretching` | `Auto` (по контенту) или `Fixed` (фиксированная ширина) |
| `ContentLeft`, `ContentRight` | Иконки слева/справа |
| `Disabled`, `Loading` | Состояния |

Поставьте `View=Accent` — кнопка станет брендового цвета. Поставьте `Disabled=True` — приглушится.

> **Важно:** на полях ввода (TextField, Select) проп `View` принимает другие значения: `Default`, `Error`, `Warning`, `Success`. Это сделано специально — кнопка-удалить и поле-с-ошибкой описываются разными словами. [Почему так →](../concepts/view-vs-validation.md)

---

## Шаг 4. Соберите форму

Соберите форму регистрации из трёх элементов:

1. Перетащите `TextField 48 M` — поле «Email».
2. Скопируйте, поставьте ниже — поле «Пароль».
3. Перетащите `BasicButton 48 M` с `View=Accent` — кнопка «Зарегистрироваться».

Между полями оставьте **16px**, между блоком полей и кнопкой — **24px**. Это значения из шкалы отступов SDDS, кратные 4. [Полная шкала →](../reference/spacing.md)

Чтобы пометить email обязательным:

1. Выделите TextField email.
2. Поставьте `Required=True`.
3. `RequiredPlacement=Right` — звёздочка справа от лейбла.

Чтобы показать ошибку:

1. Выделите TextField email.
2. Поставьте `View=Error`.
3. В hint введите текст ошибки: «Введите корректный email».

> **Не показывайте ошибку на пустом нетронутом поле.** Ошибка — это реакция на действие пользователя, а не приветствие. [Правила валидации →](../guides/build-a-form.md)

---

## Шаг 5. Переключите тему

1. Выделите весь фрейм с формой.
2. В правой панели → **Local Variables** → найдите коллекцию `01. Theme`.
3. Переключите на другой вариант темы.

Цвета компонентов обновятся автоматически. Структура, размеры и пропы — не изменятся.

> **Если компонент не сменил цвет** — он использует захардкоженное значение вместо токена. Это баг компонента, заведите задачу в Jira. Норма — компонент адаптируется без правки слоёв.

---

## Что вы узнали

- Компоненты лежат в **Pages**, токены — в **Local Variables**.
- Размер кодируется числом и буквой: `BasicButton 48 M`. M — дефолт.
- `View` управляет семантикой компонента. Значения зависят от типа: для кнопок `Negative/Positive`, для полей `Error/Success`.
- Тема меняется подменой коллекции `01. Theme` — без правки макетов.
- Шкала отступов кратна 4: `4, 8, 12, 16, 20, 24, 28, 32, 40`.
