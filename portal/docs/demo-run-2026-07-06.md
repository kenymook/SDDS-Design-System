# Отчёт: прогон демо-сценария тремя экспертными персонами

Дата: 2026-07-06 · Прототип: https://portal-flow-prototype.vercel.app · Сценарий: `demo-scenario.md`

> ⚠️ Это симуляция: три виртуальные персоны с сеньор/лид-экспертизой проведены через прототип AI-агентом. Все действия реально выполнены в прототипе (клики, ввод, публикация), наблюдения — фактические состояния интерфейса. Симуляция калибрует сценарий и снимает очевидные барьеры, но НЕ заменяет тест с живыми людьми: у виртуальной персоны нет настоящего замешательства, усталости и привычек.

---

## Персона 1 — Марина, сеньор продуктовый дизайнер

*10 лет в продуктовом дизайне, вела редизайн мобильного банка, свободно работает с токенами и Figma Variables. Выбирает систему для нового продукта экосистемы.*

### Блок 1 — Главная (знакомство)

- Слоган «Компоненты готовы. Токены настроены. Займитесь продуктом» считывает мгновенно — «это white-label система, обещают снять с меня рутину».
- Live demo «SDDS Pay» отмечает как главный аргумент: «вижу карточки, формы и списки, которые перекрашиваются токенами — это ровно то, что я буду делать». Проверила все 5 брендов × 2 режима × 5 размеров.
- Морфинг «путь цвета» оценила профессионально: «SurfaceAccent — роль, а не значение — правильная школа, как в Material/Spectrum».
- **Чего не хватило:** галереи всех компонентов («~80» заявлено, показано ~15 в demo и спирали) и упоминания Figma-библиотеки на главной (нашла только в секции «Дизайнерам» внизу). Состояния (hover/focus/disabled) на лендинге не показаны нигде.

### Блок 2 — Настройка темы (пройдено полностью)

Фактический маршрут: Открыть DS Builder → вход (anna/demo, поля предзаполнены) → проект SberBusiness (роль Owner) → CRM Design System → Light Theme → редактор.

- Контекстная панель редактора (проект / система / тема / версия / роль / счётчик changes) — «сразу понимаю, где я и что трогаю». ✅
- Выбрала токен `primary`, открыла color picker: вкладки Custom/Library, форматы Hex/RGB/HSL — «как в Figma, ничему учиться не надо». Ввела `#D45B35`.
- Панель свойств показала режимы Light/Dark и 4 контекста (Default/OnDark/OnLight/Inverse) с auto-значениями + импакт (2 темы, 4 компонента) — «глубина как у взрослых систем».
- Changes: diff `#107f8c → #d45b35`, автор, время, «2 themes · 4 components · 17 instances», статус Passed — «наконец-то вижу, что именно уеду в прод».
- Publish: версия предзаполнена 1.5.0, changelog предзаполнен → Version 1.5.0 опубликована. Итог: **правка цвета ~8 кликов, публикация ~4 клика, без единого тупика.**

Контрольные вопросы: про изоляцию черновиков ответила верно после подсказки интерфейса (бейдж Draft/Published); способ отката нашла — но только внутри модалки «Детали» версии.

**Находки Марины:**
| # | Наблюдение | Серьёзность |
|---|---|---|
| M1 | В браузере токенов всего 2 цветовых токена — не оценить работу с реальным каталогом (сотни токенов): нужны поиск/группы | трение |
| M2 | «Восстановить как draft» спрятан в модалке «Детали» — из списка Versions действие не видно | трение |
| M3 | На лендинге нет галереи компонентов и ссылки на Figma-библиотеку в первом экране | трение |
| M4 | Чекбокс «simulate configuration save failure» на форме Publish — тестовый артефакт виден пользователю | косметика |

**Вердикт:** «Редактор увереннее, чем я ожидала от внутреннего инструмента. Для выбора системы мне не хватает витрины компонентов и её Figma-стороны». **Оценка пригодности для запуска: 8/10.**

## Персона 2 — Дмитрий, лид фронтенд-разработчик

*12 лет, React/TypeScript, микрофронтенды, строил дизайн-системную инфраструктуру. Оценивает контракт «дизайн → код» и стоимость интеграции.*

### Блок 1 — Главная

- Ищет глазами технические факты: нашёл секцию Developer handoff — `sdds sync --project ... --version 1.5.0` — «версионируемая конфигурация вместо макета — это то, за что я готов простить многое».
- Секция «Разработчикам»: React, Compose UI, ViewSystem, SwiftUI — «все наши платформы, ок».
- **Скепсис:** «CLI-контракт где посмотреть? Схема target files? По лендингу не понять, это работает или это слайд». Ссылки на репозиторий/пакет/спеку CLI нет.

### Блок 2–3 — Публикация и handoff (пройдено)

- Прошёл тот же маршрут публикации: отметил автоинкремент версии, immutable-семантику («Version нельзя перезаписать» — проверил текстом ошибки при повторе номера) и validation snapshot внутри Version — «это правильные примитивы, как у пакетных менеджеров».
- Экран результата публикации: готовая CLI-команда с кнопкой «Скопировать» — «путь от кнопки Publish до команды — один экран, отлично».
- Модалка версии: diff с v1.4.0 по токенам — «могу ревьюить изменения темы как PR».
- Вопрос сценария «чем отличается от „дизайнер скинул макет"» — ответ: «конфигурация с номером, снапшотом валидации и диффом. Это контракт, а не картинка. Доверяю — при условии, что CLI существует».

**Находки Дмитрия:**
| # | Наблюдение | Серьёзность |
|---|---|---|
| D1 | Нет спецификации CLI/схемы конфигурации (что именно генерируется для React vs Compose) — для оценки интеграции это блокер решения, хоть и заявлено «открытым вопросом» | блокер (для решения, не для прототипа) |
| D2 | Не показан процесс breaking changes: что будет, если токен удалили, а продукт на v1.4.0 | трение |
| D3 | В Versions нет фильтра/поиска — при десятках версий список станет неуправляемым | косметика (сейчас) |

**Вердикт:** «Модель Version + CLI — лучшая из виденных мной внутри корпораций. Но выбирать систему буду, когда увижу контракт CLI и стратегию breaking changes». **Оценка: 7/10 (модель — 9, подтверждённость — 5).**

## Персона 3 — Олег, владелец продукта

*Запускает B2B-продукт в экосистеме, KPI — время до MVP. Смотрит на зрелость, риски и поддержку.*

### Блок 1 — Главная

- Первый экран отвечает на его вопросы за 30 секунд: ~80 компонентов, Web/Mobile/TV, WCAG 2.1 AA, «16+ команд Сбера» с узнаваемыми логотипами — «социальное доказательство сильное, если правда».
- Витрина реальных скриншотов (GigaChat, SberDevices, видео FinAI/PlatformAI) — «убеждает лучше любых слов, но 2 плейсхолдера из 8 слегка подтачивают доверие».
- Секция «Что SDDS даёт вашей роли» → «Продукту»: быстрее запуск, консистентность, WCAG — «мой чек-лист закрыт».

### Блоки 2–3 — наблюдал за «командой»

- Смотрел, как «дизайнер» публикует тему: «от входа до публикации — минуты, без разработчиков и review. Для меня это минус одна зависимость в критическом пути».
- Отметил риск: «direct publish без review — кто отвечает, если дизайнер сломает прод-тему? Хочу опциональный approve для прод-окружений».
- Releases на портале: версии с changelog — «прозрачность есть».

**Находки Олега:**
| # | Наблюдение | Серьёзность |
|---|---|---|
| O1 | Не хватает «условий обслуживания»: SLA поддержки, каналы, процесс заведения запросов на новые компоненты | трение |
| O2 | Direct publish без review — риск для прод-тем, нужна опция согласования | трение |
| O3 | Плейсхолдеры в витрине продуктов снижают эффект social proof | косметика |
| O4 | Нет кейса с цифрами («команда X собрала MVP за N недель») — для защиты выбора перед руководством | трение |

**Вердикт:** «Продаёт лучше, чем большинство внутренних платформ. Для решения о запуске жду: SLA, политику review и один кейс с цифрами». **Оценка: 8/10.**

## Повторившиеся находки → кандидаты в спринт

1. **Откат версии не видим из списка Versions** (M2; заметили двое) — вынести «Восстановить как draft» кнопкой в строку версии.
2. **Витрина/галерея компонентов отсутствует** (M3 + вопрос Олега «что входит в 80») — вернуться к решению из бэклога (индекс в документации + секция на главной).
3. **Тестовый чекбокс на Publish** (M4) — скрыть за query-параметр или подписать явнее как отладку.
4. **CLI-контракт** (D1) — хотя это открытый вопрос продукта, на лендинг/страницу Builder стоит добавить честную плашку «контракт CLI — в проработке», чтобы не выглядело готовым.
5. **Политика review для прод-тем** (O2 + известный риск из docs) — добавить в open-questions приоритет.

## Ограничения этого прогона

- Персоны не «застревают» по-человечески: реальные тайминги и эмоциональные реакции даст только живой тест (сценарий готов, участники нужны настоящие).
- Каталог токенов в прототипе мал (2 цветовых токена) — часть выводов о редакторе экстраполирована.
- Видео прогона из среды агента не записывается; возможна запись GIF через расширение Claude in Chrome при подключённом браузере.
