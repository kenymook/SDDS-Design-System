# Модель проектов и дизайн-систем

> Статус: Draft  
> Решение: Project является границей данных и доступа, заменяя пользовательский термин Tenant.

## Иерархия

```text
User
  └── Project Membership
        └── Project
              └── Design System
                    ├── Components
                    └── Theme
                          ├── Tokens
                          ├── Modes
                          ├── SubThemes
                          ├── Changes
                          └── Versions
```

Пользователь может состоять в нескольких проектах с разными ролями. Проект содержит одну или несколько дизайн-систем, каждая дизайн-система — одну или несколько тем.

## Personal Space

После авторизации пользователь попадает в персональное пространство Builder. Это не отдельная business entity, а агрегированное представление Project Membership:

- `Мои проекты` — проекты, в которых пользователь Owner;
- `Доступные мне` — проекты, куда пользователь приглашён как Editor или Viewer.

Один и тот же пользователь может быть Owner в одном Project, Editor в другом и Viewer в третьем.

## Project

Изолированное рабочее пространство продукта или команды. Project является границей доступа, настроек и audit log.

| Поле | Тип | Описание |
|---|---|---|
| id | string | Уникальный идентификатор |
| name | string | Название проекта |
| description | string | Описание |
| status | enum | active / archived |
| ownerId | userId | Ответственный |
| designSystems | array | Дизайн-системы проекта |
| members | array | Участники и роли |

## Project Membership

Связывает пользователя с проектом и определяет его роль внутри конкретного проекта.

| Поле | Описание |
|---|---|
| userId | Пользователь |
| projectId | Проект |
| role | owner / editor / viewer |
| status | active / invited / disabled |

## Design System

| Поле | Описание |
|---|---|
| id | Уникальный идентификатор |
| projectId | Родительский проект |
| name | Название дизайн-системы |
| components | Компоненты дизайн-системы |
| themes | Темы дизайн-системы |
| currentVersion | Последняя опубликованная версия |
| repository | Ссылка на репозиторий, если применимо |
| designLibrary | Ссылка на Figma/Pixso, если применимо |
| status | active / archived |

## Theme

Тема принадлежит одной дизайн-системе и хранит токены, modes, subthemes, изменения и версии.

```text
SDDS Base Theme
  └── Design System Theme
        └── SubTheme
```

Inherited value используется по умолчанию; override должен быть явно виден и попадать в Changes.

## UX-требования

- текущие Project, Design System и Theme всегда видны;
- смена контекста предупреждает о несохранённых изменениях;
- права пересчитываются после смены проекта;
- если проектов нет, показывается empty state и Request access/Create project;
- данные разных проектов не смешиваются в поиске, истории и публикациях.

Создатель Project автоматически получает роль Owner. Приглашения Editor/Viewer можно добавить непосредственно при создании Project или позже в его Settings. Membership всегда относится только к выбранному Project; глобального списка прав на всё Personal Space нет.
