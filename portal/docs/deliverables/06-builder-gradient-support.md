# Поддержка градиентов в DS Builder

Спринт-задача · [Прототип](https://portal-flow-prototype.vercel.app) → редактор темы.

## 1. О чём задача

Добавить в Builder возможность создавать и редактировать градиенты как часть визуальных настроек темы. Градиент работает рядом с цветом: пользователь управляет значением в редакторе, видит результат в preview, а изменение сохраняется в Theme и попадает в Changes.

## 2. Что входит

| Область | Решение |
|---|---|
| Где редактируется | В редакторе темы через color picker / gradient picker |
| Что настраивается | Тип градиента, цветовые точки, позиция точки, цвет, прозрачность |
| Как проверяется | Preview обновляется сразу после изменения |
| Как сохраняется | Значение относится к Theme и учитывается как изменение |
| Как передаётся дальше | Через общий механизм Version / Published Configuration |

## 3. Пользовательский сценарий

```text
Открыть Theme Editor
  → выбрать токен или визуальное значение с поддержкой градиента
  → открыть picker
  → выбрать тип градиента
  → настроить стопы, цвета и прозрачность
  → увидеть результат в preview
  → сохранить изменение в Changes
```

## 4. Что важно не потерять в финальных макетах

- Градиент должен быть виден как отдельный тип значения, а не как обычный цвет.
- Активный stop должен быть понятен визуально.
- Добавление, удаление и редактирование stop должны быть предсказуемыми.
- Preview должен показывать итоговое значение сразу.
- Viewer видит градиент только для просмотра; редактирование доступно Editor и Owner.

## 5. Статус

Функционал реализован в прототипе/Builder и считается закрытым на уровне S14. В S15 его нужно перенести в финальный UI и handoff редактора темы.
