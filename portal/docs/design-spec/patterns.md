# Паттерны и элементы интерфейса

Повторяющиеся элементы, которые встречаются на многих экранах. Их отрисовывают один раз и переиспользуют, а в спецификациях экранов просто ссылаются сюда.

## Навигация и каркас

| Элемент | Что это | Где |
|---|---|---|
| Шапка портала | Логотип + разделы (Главная, Компоненты, DS Builder, Онбординг, Релизы, Поддержка) + «Открыть DS Builder» + меню на телефоне | публичная часть |
| Меню на телефоне | Выпадающая панель со всеми пунктами; активный подсвечен | узкий экран |
| Панель проектов | Узкая вертикальная колонка: иконки проектов, «создать», профиль, уведомления | DS Builder |
| Строка контекста | Project / Design System / Theme · версия · статус · N изменений · роль | рабочее пространство Builder |
| Хлебные крошки | Показывают, где человек находится | публичная часть и Builder |
| Подвал | Название + сброс (в бою — реальные ссылки) | публичная часть |

## Стартовый экран и восстановление сессии

Куда попадает человек при возврате — зависит от того, где он был, и закрывал ли сайт.

| Часть | Поведение при возврате | Зачем так |
|---|---|---|
| Публичная (Портал) | Новый заход (сайт закрывали, сессия завершилась) открывается на **Главной**, а не на последнем разделе. Обновление страницы внутри той же сессии оставляет на месте | Публичный сайт — каждый визит логично начинать с начала; попасть сразу на случайный глубокий раздел сбивает с толку |
| DS Builder | Всегда открывается **последний рабочий экран** — как IDE открывает последний проект | Builder — рабочий инструмент; продолжать с места, где остановился, здесь удобно |

- «Сессия» = пока открыта вкладка. Закрыл сайт → сессия завершилась → следующий вход в Портал ведёт на Главную.
- Разделение проходит по границе входа: экраны за авторизацией (Builder) восстанавливаются, публичные разделы — нет.

## Всплывающие окна

| Окно | Как открывается | Как закрывается | Заметки |
|---|---|---|---|
| Модалка сущности | Действие над сущностью (переименовать/удалить/переместить) | Esc / клик мимо / крестик | фокус запирается внутри |
| Детали версии | «Детали» у версии | так же | различия, снимок, CLI, «Восстановить как черновик» |
| Пипетка цвета | Клик по образцу цвета токена | клик мимо / крестик | вручную (значение, форматы) и из готовой палитры |
| Уведомления | Колокольчик | клик мимо | список, «прочитать всё» |
| Профиль | Аватар | клик мимо | профиль, выход |
| Меню сброса | «Сбросить» | клик мимо | «сбросить раздел» / «сбросить всё» |
| Всплывашка | Событие (публикация, доступ) | сама / крестик | короткое сообщение о статусе |

## Формы

| Элемент | Из чего |
|---|---|
| Форма-карточка | Подпись сверху, поля, кнопки внизу (Отмена / основная) |
| Приглашения при создании | Строки «имя / почта / роль (Editor или Viewer)», можно добавлять и убирать |
| Проверка | Подсветка поля + сообщение; отправка заблокирована |
| Фильтры-списки | «Все / свои / добавленные», «изменённые / созданные / А-Я / Я-А» |

## Данные и статусы

| Элемент | Что это |
|---|---|
| Метка статуса | Пройдено / предупреждение / ошибка / черновик / опубликовано — с понятной цветовой семантикой |
| Строка различий | Старое → новое (у токенов и компонентов) |
| Строка влияния | Сколько тем, компонентов и вхождений затронуто |
| Карточка сущности | Проект / Design System / тема: заголовок, статус, версия, меню «⋯» |
| Пустое состояние | Иконка + текст + основное действие |

## Компоненты SDDS в демо и предпросмотре

Живые превью (кнопка, поле, переключатель, чип, значок, вкладка, всплывашка, ползунок, выпадающий список, аватар, прогресс) в живом демо и документации собраны на демонстрационных токенах. В финале это будут настоящие компоненты SDDS из библиотеки в Figma.

## Правило

Прежде чем рисовать новый экран, проверьте — может, элемент уже есть здесь. Уникальные элементы добавляйте в этот список, а не переописывайте в каждой спецификации.
