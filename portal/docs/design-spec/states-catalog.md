# Каталог состояний

Общий список состояний экранов. В спецификации каждого экрана перечисляются те, что к нему применимы, со ссылкой сюда. Смысл простой: чтобы при отрисовке ни одно состояние не забыли.

## Универсальные состояния

| Состояние | Когда бывает | Как рисуем |
|---|---|---|
| **Обычное** | Данные есть, всё доступно | Базовый вид экрана |
| **Загрузка** | Данные ещё грузятся | Скелетоны на месте контента (а не спиннер по центру, где можно) |
| **Пусто** | Данных нет — первый вход, пустой список | Иконка или иллюстрация + объяснение + кнопка «создать / запросить» |
| **Ошибка** | Что-то не загрузилось или не сработало | Спокойное сообщение с причиной + действие (повторить / перейти) |
| **Только для просмотра** | Нет прав на правку (Viewer) | Поля недоступны для ввода, кнопки правки спрятаны или объяснены, баннер «только для просмотра» + «запросить права» |
| **Успех** | Действие завершилось | Экран или всплывашка с подтверждением + следующий шаг |
| **Действие заблокировано** | Временно нельзя | Кнопка неактивна + подсказка почему |
| **Проверка формы** | Ошибка ввода | Подсветка поля + сообщение; отправка заблокирована, пока не исправят |

## Состояния, важные для DS Builder

| Состояние | Где | Что показываем |
|---|---|---|
| **Черновик** | Тема, строка контекста | Пометка «черновик» — есть неопубликованные правки |
| **Опубликовано** | Тема, версии | Пометка «опубликовано» + номер версии |
| **Есть изменения (N)** | Строка контекста, экран изменений | Счётчик правок; ведёт к списку изменений |
| **Есть замечания (N)** | Публикация, качество, инспектор | Важность (предупреждение / ошибка / критично); не блокируют публикацию, но серьёзные требуют осознанного подтверждения |
| **Публикация не удалась** | Итог публикации | Код ошибки, идентификатор, «черновик на месте», кнопка «Повторить» |
| **Черновик устарел** | Публикация | «Сначала обновите черновик» — применить текущую опубликованную версию как основу |
| **Унаследовано / переопределено** | Редактор токенов | По умолчанию унаследовано; переопределение видно и попадает в изменения |
| **Черновики приватны** | Редактор, изменения | Правки видит только их автор; общее — только опубликованное |

## Состояния навигации и сессии

| Состояние | Где | Что показываем |
|---|---|---|
| **Не вошёл → закрытый экран** | Любой закрытый экран | Отправляем на вход |
| **Проектов нет** | Стартовый экран Builder | Пусто + создать проект / запросить доступ |
| **Уход с несохранённым** | При переходе | Диалог с подтверждением |
| **Непрочитанное уведомление** | Шапка / панель | Точка на колокольчике или иконке проекта; всплывашка при первом входе |

## Всплывающие окна — тоже состояние

Модалка, поповер, пипетка, уведомление ложатся поверх экрана. Для каждого описываем: как открывается, куда уходит фокус, как закрывается (Esc / клик мимо / крестик), что видно под ним. Подробности — в [паттернах](patterns.md).

## Правило полноты

Экран готов к отрисовке, когда в его спецификации отмечены **все применимые** состояния из этого списка и на каждое есть свой фрейм в макете.
