# Изменения

**Путь (route):** `changes` · **Часть:** DS Builder (после входа) · **Прототип:** тема → «Изменения»

## 1. Зачем экран
Показать все черновые правки перед публикацией: что именно изменилось, кто и когда, на что это влияет и нет ли замечаний. Отсюда же можно откатить правку или уйти на публикацию.

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer | Смотрит список; откатывать и публиковать нельзя |
| Editor / Owner | Откатывает свои правки, применяет готовые исправления, уходит на публикацию |

## 3. Из чего состоит
1. **Заголовок** — сколько черновых правок, напоминание «замечания не блокируют публикацию», кнопка «Перейти к публикации».
2. **Полоса статуса черновика** — если черновик устарел относительно опубликованной версии.
3. **Группы** — «Токены» и «Компоненты», в каждой список правок.
4. **Карточка правки** — важность, название, старое → новое, автор · дата · что затронуто, при необходимости пояснение; действия: готовое исправление, «к источнику», «откатить».

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Правки по группам |
| Пусто | ✓ | «Нет черновых изменений» + кнопка «Открыть редактор» |
| Только для просмотра (Viewer) | ✓ | Кнопки отката и исправления недоступны |
| Есть замечание | ✓ | Важность (предупреждение/ошибка/критично) + готовое исправление |
| Черновик устарел | ✓ | Полоса статуса с предложением обновить черновик |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Готовое исправление | есть замечание, Editor/Owner | Применяет исправление |
| К источнику | всегда | Открывает токен или компонент в редакторе |
| Откатить | Editor/Owner | Убирает правку |
| Перейти к публикации | можно публиковать | → Публикация |

## 6. Как сюда попадают и куда уходят
- **Входы:** редактор темы и компонентов, строка контекста (счётчик изменений), уведомление.
- **Выходы:** редактор (к источнику), публикация.

## 7. Тексты
Названия сущностей — в оригинале. Строка «старое → новое» показывает конкретные значения. Формулировки замечаний — с понятной рекомендацией.

## 8. Телефон
Список в одну колонку; действия карточки переносятся под содержимое.

## 9. Доступность
Карточки и кнопки доступны с клавиатуры; важность передаётся не только цветом, но и текстом.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (правки по группам)
- [ ] Пусто (нет изменений)
- [ ] С замечаниями (предупреждение, ошибка, критично) и готовым исправлением
- [ ] Черновик устарел (полоса статуса)
- [ ] Viewer (действия недоступны)
- [ ] Телефон

## 11. Что ещё не решено
Как показывать очень длинные списки правок (группировка, свёртка) — заложить в дизайн.
