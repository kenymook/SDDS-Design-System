# Публикация

**Путь (route):** `publish` · **Часть:** DS Builder (после входа) · **Прототип:** тема → «Publish»

## 1. Зачем экран
Опубликовать правки как новую версию. Дизайнер делает это сам, без согласований. Перед подтверждением он видит, что публикует, сколько замечаний и под каким номером выйдет версия.

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer | Публиковать нельзя; видит объяснение и может запросить права |
| Editor / Owner | Заполняет версию и описание, подтверждает и публикует |

## 3. Из чего состоит
1. **Заголовок** — «Прямая публикация, согласования не требуются».
2. **Сводка** — тема, число правок, число замечаний.
3. **Форма** — номер версии (подставляется следующий), описание изменений.
4. **Подтверждение риска** — галочка «понимаю риски», если есть замечания, с пояснением, что они сохранятся в снимке.
5. **Снимок проверок** — список правок с важностью, значениями и рекомендациями.
6. **Отладка прототипа** (свёрнуто) — переключатель «симулировать сбой» только для демонстрации сценария ошибки.

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Форма готова к публикации |
| Только для просмотра (Viewer) | ✓ | Публикация недоступна, объяснение + «запросить права» |
| Черновик устарел | ✓ | «Сначала обновите черновик» + действие |
| Публиковать нечего | ✓ | «Нет правок» + кнопка в редактор |
| Есть замечания | ✓ | Галочка подтверждения риска обязательна |
| Ошибка публикации | ✓ | Сообщение об ошибке над формой (сам итог — на следующем экране) |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Опубликовать | форма заполнена, риск подтверждён | Создаёт версию → экран итога |
| Подтвердить риск | есть замечания | Разблокирует публикацию |
| Обновить черновик | черновик устарел | Применяет опубликованную версию как основу |
| Отмена | всегда | Назад к изменениям |
| Запросить права | Viewer | Запрос владельцам |

## 6. Как сюда попадают и куда уходят
- **Входы:** изменения, редактор, версии.
- **Выходы:** итог публикации, изменения (отмена).

## 7. Тексты
Пояснение про замечания честное: они не блокируют публикацию, но сохраняются в снимке и будут видны разработчику. Отладочный переключатель подписан прямо как отладка.

## 8. Телефон
Форма и снимок в одну колонку.

## 9. Доступность
Обязательные поля и галочка риска корректно помечены; ошибки формы объяснены текстом.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (форма)
- [ ] С замечаниями (галочка риска)
- [ ] Черновик устарел
- [ ] Публиковать нечего
- [ ] Viewer (недоступно)
- [ ] Телефон

## 11. Что ещё не решено
Нужно ли согласование для боевых тем (сейчас нет) — открытый вопрос после прогонов с пользователями.
