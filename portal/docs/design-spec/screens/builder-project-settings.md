# Настройки проекта

**Путь (route):** `project-settings` · **Часть:** DS Builder (после входа) · **Прототип:** проект → настройки (только Owner)

## 1. Зачем экран
Управление проектом целиком: название, участники и роли, запросы доступа, приглашения, журнал действий и удаление проекта. Это «панель хозяина».

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer / Editor | Нет доступа — показываем экран «нет доступа» |
| Owner | Полный доступ |

## 3. Из чего состоит
1. **Заголовок** — «Настройки проекта» + название, кнопка «Готово».
2. **Основное** — поле «Название».
3. **Иконка/обложка** — оформление проекта.
4. **Участники и доступ** — таблица: имя, почта, статус, выбор роли, удаление. Последнего Owner нельзя понизить или удалить (эти элементы заблокированы с пояснением).
5. **Запросы доступа** — если кто-то запросил права: назначить Editor или отклонить.
6. **Пригласить участника** — имя, почта, роль (только Editor или Viewer).
7. **Журнал действий** — история событий проекта.
8. **Удалить проект** — опасная зона с вводом названия для подтверждения.

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Все разделы настроек |
| Нет доступа (не Owner) | ✓ | Экран «нет доступа» |
| Есть запросы доступа | ✓ | Блок запросов с кнопками решения |
| Нет запросов | ✓ | Блок запросов не показываем |
| Последний Owner | ✓ | Его роль и удаление заблокированы с пояснением |
| Пустой журнал | ✓ | «Пока нет записей» |
| Проверка / ошибка | ✓ | Ошибки в полях и над формами |
| Подтверждение удаления | ✓ | Удаление только после ввода точного названия |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Сохранить название | заполнено | Обновляет проект |
| Сменить роль участника | не последний Owner | Меняет роль (участник получает уведомление) |
| Удалить участника | не последний Owner | Убирает из проекта |
| Решить запрос доступа | есть запрос | Назначает Editor или отклоняет |
| Пригласить | заполнено | Добавляет участника (Editor/Viewer) с уведомлением |
| Удалить проект | введено название | Удаляет проект и его содержимое |
| Готово | всегда | Назад к обзору проекта |

## 6. Как сюда попадают и куда уходят
- **Входы:** обзор проекта (кнопка настроек).
- **Выходы:** обзор проекта.

## 7. Тексты
Подчёркиваем: участие действует только внутри этого проекта. Опасная зона отделена по тону и виду; удаление защищено вводом названия.

## 8. Телефон
Таблица участников превращается в карточки-строки; формы в одну колонку.

## 9. Доступность
Заблокированные элементы (последний Owner) объяснены текстом, а не только визуально; опасные действия требуют осознанного ввода.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (все разделы)
- [ ] Нет доступа (не Owner)
- [ ] Запросы доступа (есть/нет)
- [ ] Последний Owner (заблокировано)
- [ ] Приглашение и его проверка
- [ ] Опасная зона: удаление
- [ ] Телефон (таблица → карточки)

## 11. Что ещё не решено
Права на журнал действий (сейчас его видят Editor и Owner) — подтвердить с безопасностью.
