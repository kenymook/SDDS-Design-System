# Редактор темы

**Путь (route):** `editor` · **Часть:** DS Builder (после входа) · **Прототип:** войти `ivan`/`demo` → CRM Design System → Light Theme

## 1. Зачем экран
Рабочее место, где настраивают тему: правят токены (цвет, градиенты, шрифты, скругления) с живым предпросмотром, проверяют качество, и правки попадают в список изменений. Это сердце DS Builder.

## 2. Кто видит и что может
Только вошедшие. Поведение зависит от роли (см. [правила RBAC](../../deliverables/03-rbac-ui-rules.md)).

| Роль | Что может |
|---|---|
| Viewer | Смотрит значения; править нельзя; баннер «только для просмотра» + «запросить права» |
| Editor | Полностью правит токены, смотрит предпросмотр, проверки, откат, отмену/повтор |
| Owner | То же, что Editor |

## 3. Из чего состоит
1. **Строка контекста** — Project / Design System / Theme / версия / статус / N изменений / роль.
2. **Панель инструментов** — имя редактируемого токена, отмена/повтор, вкладки «обзор» и «доступность», статус качества, кнопка сброса, «Опубликовать · N».
3. **Список токенов** (слева) — токены по группам (цвет / шрифты / скругления), поиск.
4. **Предпросмотр** (в центре) — реальные компоненты на текущих правках; режим «доступность» показывает контраст.
5. **Панель свойств** (справа) — значения токена: светлый и тёмный режимы, четыре контекста (обычный, на тёмном, на светлом, инверсия), унаследовано или переопределено, влияние (темы / компоненты / вхождения), переходы к компонентам.
6. **Пипетка / gradient picker** (всплывает) — по клику на образец цвета или градиентное значение.

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Токен выбран, значения и предпросмотр |
| Загрузка | ✓ | Скелетоны колонок при заходе |
| Пусто | ✓ | Токен не выбран → подсказка выбрать в списке |
| Ошибка | ✓ | Недопустимое значение токена (проверка) |
| Только для просмотра (Viewer) | ✓ | Поля недоступны, правка спрятана, баннер + «запросить права» |
| Действие заблокировано | ✓ | Отмена/повтор неактивны без истории; «Опубликовать · 0» неактивна без правок |
| Проверка | ✓ | Инспектор: предупреждение или ошибка с готовым исправлением в один клик; статус «N замечаний / замечаний нет» |
| Унаследовано / переопределено | ✓ | Пометка у унаследованного; переопределение видно и уходит в изменения |
| Пипетка открыта | ✓ | Всплывающее окно выбора цвета (вручную и из палитры) |
| Gradient picker открыт | ✓ | Настройка типа градиента, stop-значений, цвета, прозрачности и позиции |
| Режим доступности | ✓ | Предпросмотр с расчётом контраста и предупреждениями |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Выбрать токен | всегда | Значения загружаются в панель свойств |
| Изменить значение / открыть пипетку или gradient picker | если можно править (Editor/Owner) | Правка → запись в изменения |
| Отмена / повтор | есть история | Откат или возврат правки |
| Готовое исправление | есть замечание | Применяет исправление |
| Откат | если можно править | Сброс правки |
| Сбросить раздел / всё | если можно править | Сброс изменений |
| Вкладка «доступность» | всегда | Режим контраста |
| «Опубликовать · N» | можно публиковать, N>0 | → Публикация |
| «Запросить права» | Viewer | Запрос уходит владельцам |
| Перейти к компоненту | всегда | Переход к затронутому компоненту |

## 6. Как сюда попадают и куда уходят
- **Входы:** обзор Design System → тема; обзор темы; уведомление; онбординг дизайнера.
- **Выходы:** изменения, публикация, редактор компонентов, качество, назад к Design System.

## 7. Тексты
Названия сущностей и токенов — в оригинале (SurfaceAccent, CRL, orange-500). Сообщения проверок — понятные, с предложением, как исправить.

## 8. Телефон
Три колонки — это десктопный сценарий. На узком экране панель свойств прячется, редактор сводится к списку токенов и предпросмотру; полноценная работа предполагается на десктопе.

## 9. Доступность
- Пипетка и gradient picker: ползунки и stop-значения управляются с клавиатуры, фокус заперт внутри окна.
- Режим «доступность» — это и есть инструмент проверки контраста.
- Строки токенов и вкладки доступны с клавиатуры.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное состояние (Editor, токен выбран)
- [ ] Токен не выбран (пусто)
- [ ] Пипетка открыта (вручную и из палитры)
- [ ] Gradient picker открыт: тип, preview, список stop-значений, активный stop, цвет, opacity, remove/add stop
- [ ] Панель свойств: светлый/тёмный + четыре контекста, унаследовано против переопределено
- [ ] Проверка: предупреждение и ошибка с готовым исправлением
- [ ] Режим доступности (предпросмотр с контрастом)
- [ ] Viewer: только просмотр + баннер «запросить права»
- [ ] Отмена/повтор неактивны против активных
- [ ] Узкий экран (панель свойств спрятана)

## 11. Что ещё не решено
- Точный перечень изменяемых / ограниченных / закрытых свойств — открытое решение (влияет на редактор компонентов и блокировки).
- Как ведёт себя список токенов при их сотнях (поиск, группы, подгрузка) — заложить в дизайн.
