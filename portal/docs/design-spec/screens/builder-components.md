# Редактор компонентов

**Путь (route):** `components` · **Часть:** DS Builder (после входа) · **Прототип:** тема → вкладка компонентов

## 1. Зачем экран
Настройка отдельных компонентов темы в разрешённых рамках. У каждого свойства своя политика: что-то можно менять свободно, что-то — в пределах, а что-то закрыто и доступно только для просмотра. Здесь же видно, какие токены связаны с компонентом.

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer | Смотрит свойства; править нельзя; баннер «только для просмотра» + «запросить права» |
| Editor / Owner | Правит разрешённые свойства; ограниченные — в пределах; закрытые не трогает |

## 3. Из чего состоит
1. **Список компонентов** (слева) — с поиском; у каждого метка политики (можно менять / ограничено / закрыто) и число компонентов.
2. **Панель инструментов** — имя выбранного компонента, отмена/повтор, вкладки «свойства» и «состояния», переход к изменениям и публикации.
3. **Панель свойств** (справа) — политика выбранного свойства с пояснением; поле правки (недоступно для Viewer и для закрытых); что затронуто (темы, вхождения); связанные токены с переходом к ним.
4. **Предпросмотр** (в центре) — компонент с текущим значением; в режиме «состояния» — его интерактивные состояния.

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Компонент выбран, свойства и предпросмотр |
| Пусто (поиск) | ✓ | «Ничего не найдено» в списке и на предпросмотре |
| Только для просмотра (Viewer) | ✓ | Поле недоступно, баннер + «запросить права» |
| Свойство закрыто | ✓ | Поле недоступно, пояснение «управляется SDDS» |
| Свойство ограничено | ✓ | Пояснение допустимого диапазона |
| Действие заблокировано | ✓ | Отмена/повтор неактивны без истории; публикация неактивна без правок |
| Есть правка | ✓ | Кнопка «Сбросить» у изменённого свойства |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Поиск компонента | всегда | Фильтрует список |
| Выбрать компонент | всегда | Загружает свойства и предпросмотр |
| Изменить свойство | Editor/Owner, свойство не закрыто | Правка → в изменения |
| Сбросить | есть правка | Откат свойства |
| Вкладка «состояния» | всегда | Предпросмотр интерактивных состояний |
| Перейти к связанному токену | всегда | Открывает токен в редакторе |
| Отмена / повтор, изменения, публикация | по условиям | Соответствующие действия/экраны |

## 6. Как сюда попадают и куда уходят
- **Входы:** редактор темы, обзор Design System, уведомление.
- **Выходы:** изменения, публикация, редактор токенов (через связанный токен).

## 7. Тексты
Названия компонентов и свойств — в оригинале. Пояснения политик короткие и понятные («Можно изменять без ограничений» / «Допустимый диапазон: …» / «Управляется SDDS, только просмотр»).

## 8. Телефон
Три колонки — десктопный сценарий; на узком экране панель свойств прячется.

## 9. Доступность
Строки компонентов и вкладки доступны с клавиатуры; поле правки корректно помечается как недоступное для скринридера.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (Editor, компонент выбран)
- [ ] Три политики свойства: можно / ограничено / закрыто
- [ ] Вкладка «состояния» (интерактивные состояния)
- [ ] Пусто (поиск ничего не нашёл)
- [ ] Viewer: только просмотр + баннер
- [ ] Узкий экран (панель свойств спрятана)

## 11. Что ещё не решено
Точный перечень изменяемых / ограниченных / закрытых свойств — открытое решение команды системы. От него зависит и этот экран, и правила блокировок.
