# Настройки темы

**Путь (route):** `theme-settings` · **Часть:** DS Builder (после входа) · **Прототип:** тема → настройки

> Уточнение по прототипу: этот экран — именно **настройки** темы (название, обложка, перенос, удаление), а не дашборд-обзор. Обзорная информация о теме (статус, версия, изменения, качество) живёт в строке контекста и в списке тем на экране Design System.

## 1. Зачем экран
Управлять самой темой как сущностью: переименовать, задать обложку, перенести в другую Design System или удалить. Это административные действия, отдельные от редактирования содержимого темы.

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer | Нет доступа — показываем экран «нет доступа» |
| Editor / Owner | Полный доступ к настройкам темы |

## 3. Из чего состоит
1. **Заголовок** — «Настройки темы» + название, кнопка «Готово» (назад к Design System).
2. **Основное** — поле «Название», сохранение.
3. **Обложка** — управление обложкой темы.
4. **Перенести тему** — выбор целевой Design System (из доступных, где роль Owner или Editor), кнопка «Перенести».
5. **Удалить тему** — опасная зона: ввод названия для подтверждения, кнопка удаления.

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Формы настроек |
| Нет доступа (Viewer) | ✓ | Экран «нет доступа» |
| Проверка формы | ✓ | Ошибки в полях (например, пустое название) |
| Ошибка настроек | ✓ | Сообщение об ошибке над формами |
| Нельзя перенести | ✓ | Кнопка «Перенести» неактивна, если нет подходящей целевой системы |
| Подтверждение удаления | ✓ | Удаление доступно только после ввода точного названия |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Сохранить название | заполнено | Обновляет название темы |
| Задать обложку | всегда | Меняет обложку |
| Перенести | есть целевая система | Переносит тему в другую Design System |
| Удалить | введено точное название | Удаляет тему (с подтверждением) |
| Готово | всегда | Назад к Design System |

## 6. Как сюда попадают и куда уходят
- **Входы:** обзор Design System (меню темы), строка контекста.
- **Выходы:** Design System (готово, после переноса или удаления).

## 7. Тексты
Опасная зона (удаление) визуально и по тону отделена; требование ввести название защищает от случайного удаления.

## 8. Телефон
Формы в одну колонку.

## 9. Доступность
Опасные действия требуют осознанного ввода; ошибки объяснены текстом.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (формы настроек)
- [ ] Нет доступа (Viewer)
- [ ] Перенос недоступен (нет целевой системы)
- [ ] Опасная зона: удаление с подтверждением
- [ ] Телефон

## 11. Что ещё не решено
Нет открытых вопросов по этому экрану сверх общих.
