# Главная

**Путь (route):** `portal-home` · **Часть:** публичная · **Прототип:** [portal-flow-prototype.vercel.app](https://portal-flow-prototype.vercel.app)

## 1. Зачем экран
Первое знакомство с SDDS. Новой команде — понять, что это, убедиться и оставить заявку. Действующей — быстро уйти в Builder из шапки. Экран ведёт человека вниз по рассказу.

## 2. Кто видит и что может
Открыт всем, без входа. Вариантов по ролям нет.

## 3. Из чего состоит (сверху вниз)
1. **Шапка** — навигация + «Открыть DS Builder».
2. **Первый экран** — движущаяся лента компонентов на фоне, заголовок, слоган, подзаголовок, две кнопки, четыре цифры (~80 компонентов · Web·Mobile·TV · размерная шкала · WCAG 2.1 AA).
3. **Доказательство** — «16+ команд Сбера…», сетка продуктов.
4. **Витрина продуктов** — блок со скриншотами и видео, который вырастает почти на весь экран, слайды меняются сами.
5. **Принципы** — четыре карточки с мини-иллюстрациями.
6. **Иконки** — три начертания (тонкое, жирное, заливка) и три размера (16/24/32).
7. **Как устроены токены** — анимация «путь цвета» + текст.
8. **Живое демо** — интерактив: бренд / режим / размер.
9. **DS Builder** — короткий мостик + макет, кнопка «Подробнее».
10. **Передача разработчику** — команда CLI (+ пометка «в проработке»).
11. **AI-ready** — MCP (бета), макет чата, четыре пункта.
12. **Что даёт по ролям** — три карточки.
13. **Финальный призыв** — «Начать знакомство» / «Запросить доступ».

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Весь лендинг |
| Загрузка | ✓ | Секции проявляются анимацией; фон первого экрана — запасной кадр, если техника не поддерживается |
| Пусто | — | — |
| Ошибка | ✓ | В витрине: если картинка слайда не загрузилась — заглушка с названием продукта |
| Только для просмотра | — | — |
| Успех | — | — |
| Меньше движения | ✓ | Все анимации (спираль, витрина, «путь цвета», иконки) заменяются статичными кадрами |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| «Записаться на демо» (основная) | всегда | → Поддержка, тип «Демо или консультация» |
| «Узнать больше» (вторая) | всегда | плавная прокрутка к принципам |
| «Открыть DS Builder» (шапка) | всегда | → Вход (или сразу в Builder, если уже вошёл) |
| Пункты навигации | всегда | соответствующие разделы |
| Бренд / режим / размер в живом демо | всегда | перекраска и масштаб демо-интерфейса |
| Точки и автосмена витрины | всегда | смена слайда |

## 6. Как сюда попадают и куда уходят
- **Входы:** прямая ссылка, пункт «Главная», логотип.
- **Выходы:** Поддержка (демо), принципы (прокрутка), Вход/Builder, любой раздел навигации, страница про Builder.

## 7. Тексты
Формулировки — утверждённые, из прототипа (см. [тексты и глоссарий](../content-and-glossary.md)). Слоган, подзаголовок, принципы, кнопки — не менять без согласования.

## 8. Телефон
- Узкий экран: навигация сворачивается в меню; лента первого экрана отдаляется, чтобы не наезжать на заголовок; секции в одну колонку; карточки стопкой.
- Витрина и живое демо сохраняют логику.

## 9. Доступность
- Контраст текстов проходит WCAG AA (проверено).
- Декоративные анимации спрятаны от скринридеров; тексты понятны и без них.
- Кнопки и навигация доступны с клавиатуры.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Десктоп, обычное состояние (весь лендинг посекционно)
- [ ] Первый экран: обычный и в режиме «меньше движения» (статичный фон)
- [ ] Витрина: маленький размер, максимум, заглушка слайда
- [ ] Живое демо: 2–3 бренда × светлый/тёмный × размер
- [ ] Телефон: первый экран, открытое меню, ключевые секции стопкой

## 11. Что ещё не решено
- Ставить ли цифру «AI-ready» в первый экран (пока нет — MCP в бете, чтобы не переобещать).
- Сколько иконок в наборе (для текста секции про иконки).
- Реальные скриншоты для двух оставшихся слайдов витрины (Нетология, СберСтрахование).
