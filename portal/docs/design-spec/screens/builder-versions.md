# Версии

**Путь (route):** `versions` · **Часть:** DS Builder (после входа) · **Прототип:** тема → «Versions»

## 1. Зачем экран
История опубликованных версий: что и когда вышло, чем версии отличаются, как забрать любую через CLI и как вернуть её значения в работу.

## 2. Кто видит и что может
| Роль | Что может |
|---|---|
| Viewer | Смотрит список и детали, копирует команду CLI |
| Editor / Owner | То же + «Восстановить как черновик»; для неудачных публикаций — «Повторить» |

## 3. Из чего состоит
1. **Заголовок** — «Неизменяемые опубликованные конфигурации», кнопка «Опубликовать черновик» (если есть что публиковать).
2. **Список версий** — по каждой: номер, дата, описание, статус (опубликовано / не удалось), число замечаний.
3. **Действия у строки** — «Детали», «Восстановить как черновик» (у опубликованных, для Editor/Owner), «Копировать CLI», «Повторить» (у неудачных).
4. **Детали версии** (всплывающее окно) — различия с предыдущей опубликованной, снимок, команда CLI, «Восстановить как черновик».

## 4. Состояния
| Состояние | Есть? | Что показываем |
|---|:--:|---|
| Обычное | ✓ | Список версий |
| Пусто | ✓ | Ещё нет опубликованных версий |
| Строка «не удалось» | ✓ | Статус ошибки + «Повторить» |
| Только для просмотра (Viewer) | ✓ | Нет «Восстановить как черновик» и «Повторить» |
| Детали открыты | ✓ | Всплывающее окно с различиями и действиями |

## 5. Действия и переходы
| Действие | Когда | Что происходит |
|---|---|---|
| Детали | всегда | Открывает окно с различиями и снимком |
| Восстановить как черновик | Editor/Owner, версия опубликована | Возвращает значения версии в черновик (с подтверждением); история не переписывается |
| Копировать CLI | всегда | Копирует команду получения версии |
| Повторить | Editor/Owner, публикация не удалась | Повторная публикация |
| Опубликовать черновик | есть правки | → Публикация |

## 6. Как сюда попадают и куда уходят
- **Входы:** тема, итог публикации, релизы, уведомление.
- **Выходы:** публикация, детали версии (окно).

## 7. Тексты
Статусы — «опубликовано» / «не удалось». Восстановление подчёркивает, что история не переписывается, а создаётся новый черновик.

## 8. Телефон
Строки версий переносят действия под содержимое; окно деталей — на весь экран.

## 9. Доступность
Статус передаётся текстом; окно деталей запирает фокус, закрывается с клавиатуры.

## 10. Что отрисовать (чек-лист макетов)
- [ ] Обычное (список)
- [ ] Пусто (нет версий)
- [ ] Строка «не удалось» + «Повторить»
- [ ] Окно деталей (различия, снимок, CLI, восстановление)
- [ ] Viewer (без восстановления/повтора)
- [ ] Телефон
