## Наблюдения по разделам

### Страница DS Builder (Олег)
Секции: 4 шага → проверки качества → роли → Versions/откат → CTA. Плашка «CLI-контракт в проработке» на месте (исправление №4 прогона 1 — подтверждено). «Страница отвечает на вопрос „что мой дизайнер сможет сам" — этого хватало на главной первой ревизии, теперь есть куда послать за деталями».

### Документация (Марина)
15 страниц в боковой навигации: foundations (Introducing, Changelog, Tokens, Theming, Typography…) и Components по категориям. Страница Button: хлебные крошки `Components / Actions / Button`, живое превью, 4 свойства read-only, переход «Открыть в Builder». «Спецификация компонента с живым превью — то, чего мне не хватало на главной; жаль, компонентов в доке пока меньше, чем в живой библиотеке».

### Онбординг (Марина + Дмитрий)
Трек Designer: шаги со статусами (✓/◐/○), кнопка «Шаг выполнен · дальше» работает, прогресс сохранился при смене трека и возврате. Трек Developer: «Начало работы → Найти опубликованную Version → Получить конфигурацию через CLI» — «маршрут написан от моей задачи, а не от устройства системы. Редкость».

### Релизы (Дмитрий)
Опубликованные Versions четырёх систем + SDDS Core 3.12.0 + честная карточка «CLI contract: открытые решения». «Version в релизах совпадает с тем, что я опубликовал в Builder — сквозная согласованность данных в прототипе».

### Поддержка (Олег)
Форма: 4 типа запроса, контекст предзаполнен (`SberBusiness / CRM Design System`), после отправки — подтверждение и «Создать ещё». «Каналу не хватает ожиданий по срокам (SLA) — прежняя находка O1 остаётся в силе».

### Builder: негативный путь публикации (Дмитрий)
Симуляция сбоя (теперь корректно спрятана в «Отладка прототипа» — исправление №3 подтверждено): экран Failed содержит error class `CFG-STORE-503`, Request ID, явное «Draft-изменения сохранены и не потеряны», кнопка Retry → повтор успешен, Version 1.5.0 Published. «Обработка сбоя честнее, чем у многих прод-систем: класс ошибки, request id, гарантия сохранности черновика».

### Откат (Марина)
Кнопка «Восстановить как draft» теперь в строке списка Versions (исправление №1 подтверждено): клик на v1.4.0 → подтверждение → 1 change в черновике, переход в Changes. «Раньше пришлось бы догадаться открыть „Детали" — теперь путь очевиден».

### Viewer (Олег за Марию)
Вход maria/demo: роль Viewer в SberBusiness, в теме — баннер «Read-only access» с «Request edit access», Publish заблокирован. «Ролевая модель не декорация: ограничения реально работают и объясняются, а не молча дизейблятся».
