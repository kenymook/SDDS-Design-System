# Лестница качества: от 7 к 10

Оценка прототипа SDDS Portal / DS Builder по итогам раунда 1 интервью (07.2026) и полного прохода по 44 экранам — и путь наверх. Оценка «10/10» здесь означает не «идеально», а «не осталось известных причин снимать баллы».

## Текущие оценки

| Критерий | Оценка | Одной строкой |
|---|:---:|---|
| Польза | 8.5 | Ценность реальная и подтверждённая интервью |
| Считывание контекста | 8 | Строка контекста, статусы, RBAC-объяснения — сильная черта |
| Удобство | 7 | Базовые пути проходятся без обучения; 2–3 ямы |
| Интуитивность | 6.5 | Концепт ясен, ментальная модель редактора — нет |
| Честность и доверие | 9 | «В проработке» помечено, сбои не теряют работу — фирменная черта |
| Консистентность языка | 8 | Единый словарь Project / Theme / Version / Changes |
| Позиционирование | 7 | После правок раунда 1 (было 4–5) |
| Производительность | 7 | После оптимизаций; на целевом железе не перепроверено |
| Доступность | 6 | Заявлено и частично сделано; аудита не было |
| Готовность к handoff | 5 | Держат незакрытые решения инженерии, не дизайн |

**Интегрально: ~7/10.** Слабые места локализованы и оприоритезированы — путь наверх известен.

## Уровень 1 — известные правки прототипа (7 → ~8)

| Разрыв | Действие | Что чинит |
|---|---|---|
| Интуитивность | **Palette view**: brand/neutral/status палитры → каскад в семантические токены; индикатор Light/Dark, двойное превью | Главный ментальный разрыв дизайнера |
| Удобство | **Откат**: понятная формулировка + прятать restore, когда отличий нет | Единственное проваленное задание теста |
| Интуитивность | **Документация по Storybook-паттерну**: дерево слева; на странице компонента — props, states, code, examples | Ожидания дизайнера и фронтендера |
| Удобство | **Реальные URL**: работает «Назад», открытие в новой вкладке, ссылка на экран | Самая эмоциональная жалоба фронтендера |
| Контекст | Пометить **демо-данные и владельца**; Releases «мои» по умолчанию; прогресс онбординга за авторизацию | Хвосты «чьё это?» |

## Уровень 2 — решения вне дизайна (~8 → ~9)

1. **Контракт CLI** с инженерией: команды, формат конфигурации, версионирование. Превращает dev-страницу из плейсхолдеров в правду.
2. **Перечень изменяемых props компонентов** — без него редактор компонентов не домакетировать.
3. **Реальные пакеты / GitHub** (или приватный registry) — Installation перестаёт быть витриной.

Эти переговоры стоит начинать первыми — они самые долгие.

## Уровень 3 — качество исполнения (~9)

- Финальный визуал в Figma по спекам (материалы подготовлены: 44 кадра + аннотации на странице «СС»).
- Полный аудит доступности: клавиатура во всём Builder, тултипы не только с hover, контраст второстепенных текстов, скринридер.
- Перф на целевом железе (Intel MacBook) + бюджет производительности как требование к финальной реализации.
- Онбординг, который ведёт: ссылки из доки в разделы Builder, первый сценарий «за руку».

## Уровень 4 — последний балл не делается, а зарабатывается

- **Раунд 2 юзертестов** на исправленной версии: handoff/CLI с 2–3 разработчиками, палитра с 2–3 дизайнерами. 10 по интуитивности = «новый пользователь проходит сценарии без единого вопроса».
- **Пилот с реальной командой**: настоящий проект публикует тему, разработчик забирает через CLI в реальный продукт.
- Итерация по результатам: на трёх респондентах ловится крупное, тонкое — на десяти.

> **Критерий 10/10: команда пилота публикует вторую версию темы без нашего участия.**
